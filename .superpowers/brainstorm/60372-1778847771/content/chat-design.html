App Design — LinguaChat MVP

Core architecture and screens. Let me know if anything needs adjusting.

1. Accounts & Setup

  • Sign up with email + password
  • Pick interface language (what you type in / UI language)
  • Pick language to learn
  • Invite partner via link or username
  • Partner accepts → shared chat room created

2. Message Storage

Each message stores exactly 2 things:

  • Original text — as typed, never modified
  • Translation — into partner's learning language (generated once on send)

3. What Each User Sees

Nastia's view (learns Tamil, interface: English)
நான் உன்னை நேசிக்கிறேன்
I love you
You (original)
Я тебе кохаю
I love you
Aswin (translation to Tamil → Ukrainian shown to him)

* Nastia sees her own message in Tamil (learning) + English (original she typed). Aswin's message shows in Ukrainian (his translation) + English (his original).

4. Tap-a-Word Popup

  • Tap any word in any message
  • Popup shows: word → translation in interface language + transliteration + audio button
  • Works on both the learning-language line and original line

5. Translation Toggle

  • Toggle in chat header: Show translations ON/OFF
  • OFF = only learning-language line visible (challenge mode)
  • Per-chat setting, persists across sessions

6. Input Mode

  • Default: type in interface language → auto-translated on send
  • Button next to input field: switch to "Write in [learning language]" mode
  • In learning mode: message sent as-is, reverse-translated back to partner's interface language for their subtitle

Out of scope for v1

  • Group chats
  • Voice messages
  • Word saving / vocabulary lists
  • Push notifications (basic ones only)
  • Multiple partners / chat rooms
