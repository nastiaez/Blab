B — One screen, tab toggle (Recommended)

Single screen with a pill toggle at top: [Sign up] [Log in]. No back button, no navigation — just switching modes. Name field appears/disappears. Cleanest of all — no page transitions, no confusion about where you are.

C — SSO-first, email collapsed

Default shows only Apple + Google. "Use email instead" expands a single email field. Then Parlo detects: existing account → ask password. New email → ask name + password. One flow, no sign up / log in split at all.

B removes the back button problem entirely. C is most radical but requires backend email-detection logic.
